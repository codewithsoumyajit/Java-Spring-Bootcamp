## Warm-up Questions

### 🧩 The "What's in the Box?" Category
1.  **The Acronym Test:** If you want to *write* a new Java program, do you need the **JDK** or just the **JRE**? Why?
2.  **The Engine Room:** Which component is responsible for actually taking your code and running it on the hardware? (Hint: It’s the "Machine" part of the trio).
3.  **The Translator:** We saw a tool called `javac` in the `bin` folder. What does the "c" stand for, and what is its main job?

---

### 🌐 The "Ecosystem & Versions" Category
4.  **The "Flavor" Question:** We installed **Temurin (Adoptium)**. If your future boss asks you to use **Amazon Corretto** instead, will your Java code still work? Why or why not?
5.  **The LTS Logic:** Why do most professional developers prefer an **LTS** version (like Java 17 or 21) over the very latest version that comes out every 6 months?
6.  **The Price Tag:** Is OpenJDK "Free as in speech" (open source) or just "Free as in beer" (no cost)?

---

### 💻 The "System & Troubleshooting" Category
7.  **The "Path" Puzzle:** If you type `java` in your command prompt and it works, but `javac` says "not recognized," what is likely missing from your Environment Variables?
8.  **The JAVA_HOME Mystery:** Some tools (like Maven) look for a variable called `JAVA_HOME`. Does this variable point to the `/bin` folder or the main JDK installation folder?
9.  **The Version Check:** What is the exact command you run to prove to someone that Java is installed and working correctly on your machine?

---

### 💡 The "Brain Teaser" (Critical Thinking)
10. **The Universal Language:** Java's slogan is "Write Once, Run Anywhere." Based on the session, which component (**JDK, JRE, or JVM**) is the "magic" that makes the code run on Windows, Mac, and Linux without changing the source code?
